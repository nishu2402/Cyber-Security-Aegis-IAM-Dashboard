Intel Retrieval Lab

How to export real-world IAM JSON for analysis (AWS / Azure / GCP).
Use only in authorized environments.
Back

AWS (Recommended)

  1. Install & configure AWS CLI:
    aws configure
  2. Export IAM authorization details:
    aws iam get-account-authorization-details --output json > iam_auth.json
  3. Optional: export credential report:
    aws iam generate-credential-report && aws iam get-credential-report --query Content --output text | base64 -d > credential_report.csv
  4. Upload iam_auth.json into Aegis-IAM.
Tip: If your org uses Organizations, also evaluate SCPs:
aws organizations list-policies --filter SERVICE_CONTROL_POLICY

Azure (Entra ID / RBAC)

  1. Login:
    az login
  2. Export role assignments:
    az role assignment list --all --output json > azure_role_assignments.json
  3. Export role definitions:
    az role definition list --output json > azure_role_definitions.json
  4. Upload into Aegis-IAM (adapter support can be added).

GCP (IAM)

  1. Login:
    gcloud auth login
  2. Export project IAM policy:
    gcloud projects get-iam-policy YOUR_PROJECT_ID --format=json > gcp_iam_policy.json
